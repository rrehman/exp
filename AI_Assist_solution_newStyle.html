Add a contextual menu to your docs with one-click AI integrations for ChatGPT, Claude, Perplexity, Google AI Studio, Devin, Windsurf, and MCP tools.

The contextual menu provides quick access to AI-optimized content and direct integrations with popular AI tools. When users select the contextual menu on any page, they can copy content as context for AI tools or open conversations in ChatGPT, Claude, Perplexity, Google AI Studio, Grok, Devin, Windsurf, or a custom tool of your choice with your documentation already loaded as context.

The contextual menu is enabled per portal. Once enabled, it appears on every page of your developer portal automatically.

Menu options

The contextual menu includes several pre-built options that you can enable by adding their identifier to your configuration. Each option opens a specific AI tool with your page content already loaded as context.

How it works

When a user clicks the contextual menu button on a documentation page, APIMatic converts the current page into an AI-readable format and passes it to the selected tool. The process happens entirely in the browser — no server round-trip is needed for the copy actions.

Content processing

  • Markdown conversion — The page HTML is stripped of navigation chrome and converted to clean Markdown before it is passed to any AI tool.
  • Code blocks preserved — All code samples, request/response examples, and schema snippets are retained with their language annotations.
  • Token-aware truncation — If the page exceeds the context limit of the target model, content is intelligently trimmed from the bottom up.
  • Frontmatter injection — Page title, URL, and product name are prepended as YAML frontmatter so the AI has full provenance of the content.

Supported AI tools

  • ChatGPT — Supports GPT-4o and later. Content is passed via the ChatGPT share URL scheme.
  • Claude — Supports Claude 3.5 and later via the Anthropic web app deep-link.
  • Perplexity — Opens a focused search session with the page pre-loaded as a source.
  • Grok — Uses the xAI platform deep-link to open a conversation with context.
  • Google AI Studio — Passes content to a new Gemini prompt in AI Studio.

Configuration

Enable the contextual menu in your portal settings and choose which options to expose. You can restrict the menu to specific pages or sections, or enable it globally across all content.

Prerequisites

  • An APIMatic account with an active portal plan.
  • A published developer portal with at least one API specification imported.
  • Portal admin access to modify the portal configuration file.

Steps to enable

  • Step 1 — Open your portal's portal-config.json in the APIMatic dashboard.
  • Step 2 — Add the contextualMenu key to the features object.
  • Step 3 — List the option identifiers you want to show under the options array.
  • Step 4 — Save and republish the portal. The menu appears immediately on all pages.

Customisation

Beyond the pre-built options, you can add custom entries that point to any URL. Custom entries support a {pageUrl} and {pageMarkdown} placeholder that APIMatic replaces at runtime.

  • Custom label — Set any display name for the menu item.
  • Custom icon — Provide a URL to a 32×32 PNG or SVG icon.
  • Target URL template — Use placeholders to inject live page content into the destination URL.
  • Separator control — Add visual separators between groups of menu items.
